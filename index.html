inicio primera vez que iniciamos el proyecto

    1.- carpeta con el proyecto creada
      2.- git init -> inicializar el repositorio
        3.- git add . -> agregas los cambios realizados al stage area (área de preparación)
          4.- git commit -m "Proyecto inicial" -> crear primera versión
            5.- crear repositorio en github
              6.- git remote add origin git@github.com:NelsonRamirezC/front-m2-s4-d3-ejemplos.git
                -> se debe agregar / vincular repositorio local con repositorio remoto
                  7.- git remote -v -> verificar que quedo vinculado el repositorio de github
                    8.- (opcional - rama master) ejecutan -> git branch -M main
                      9.- git push -u origin main -> se suben los cambios a la nube (github)
                        fin primera vez que iniciamos el proyecto
                          1.- git checkout -b feature/footer -> creamos nueva rama y nos cambiamos a ella
                            2.- realizar al documento -> nuevas características o funcionalidad
                              3.- git add . -> agregamos los cambios al stage
                                4.- git commit -m "Se agrega footer al documento index.html" -> se versionan los cambios
                                  5.- git checkout main -> regresamos a la rama principal
                                    6.- git merge feature/footer -> fusionamos los cambios de la rama feature/footer al main
                                      7.- git branch -d feature/footer -> eliminar rama que ya cumplió su función
                                        8.- git push origin main -> enviamos la actualización desde el main local al main remoto
                                          inicio trabajando en el proyecto
                                            PUBLICAR PROYECTO EN GITHUB PAGES
                                              1.- git checkout -b gh-pages -> crear rama llamada gh-pages
                                                2.- git push origin gh-pages -> se envía lo que tiene la rama gh-pages local a una nueva gh-pages en github
                                                  MANTER EL PROYECTO
                                                    1.- actualizar código
                                                      2.- git add.
                                                        4.- git commit -m "mensaje"
                                                          5.- git push origin main
                                                            6.- si queremos actualizar gh-pages para ver la web con los últimos cambios
                                                              7.- git checkout gh-pages
                                                                8.- git merge main
                                                                  9.- git push origin gh-pages
